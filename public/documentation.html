4. Mobile Features

Optimized for Mobile Workflow

On mobile devices, constantly switching between the Guest List and Map View to assign arriving guests can be tedious. The mobile-optimized features below streamline the assignment process by keeping critical information accessible and reducing the need to navigate back and forth between views.

Mobile Quick Assign Panel

Available on mobile devices only (hidden on tablets and desktops)

A floating bottom panel that provides quick access to all unassigned guests while viewing the map. This eliminates the need to switch back to the List View to select a guest.

How to Use

  1. 01 While on the Map View, tap the floating users icon in the bottom-right corner to open the Quick Assign Panel.
  2. 02 Scroll horizontally through the list of unassigned guests displayed as pill-shaped buttons.
  3. 03 Each guest button shows their name and the number of seats remaining to assign (e.g., "Smith Party • 4 seats").
  4. 04 Tap a guest to select them, then tap a table on the map to assign them.
  5. 05 Tap the × button on the panel to close it.

Pro Tip: The Quick Assign Panel automatically updates as guests are assigned, showing only those who still need seating.

Selected Guest Banner

Context-aware header (mobile only)

When you select a guest, a banner appears at the top of the Map View displaying the guest's name and remaining seats. This persistent reminder helps you stay focused on the current assignment.

Features

  • Guest Information: Displays selected guest name and remaining seat count.
  • Navigation Arrows: Use Previous and Next buttons to cycle through unassigned guests without leaving the map.
  • Close Button: Tap × to deselect the current guest and dismiss the banner.

Auto-Return to List View

Smart navigation for efficient workflow

When you fully assign a guest (all their seats are allocated) on a mobile device, the app automatically switches back to the List View. This intelligent behavior helps you quickly locate and select the next arriving guest without manual navigation.

Workflow Example

1 Select "Smith Party (4 seats)" from List View
2 App switches to Map View automatically
3 Tap Table 5 to assign all 4 seats
✓ App returns to List View — ready for the next guest

Note: This feature only activates on mobile devices (screen width less than 1024px). Desktop users remain on the Map View.

Swipe Gestures

Touch-based guest navigation (mobile only)

Quickly navigate between unassigned guests using intuitive swipe gestures on the Map View. This provides a natural, touch-friendly alternative to tapping navigation buttons.

Gesture Controls

Swipe Right

Select previous unassigned guest

Swipe Left

Select next unassigned guest

Swipe Threshold: A minimum swipe distance of 50 pixels is required to trigger guest navigation, preventing accidental selections.

Availability: Swipe gestures are only active on mobile devices. Desktop and tablet users should use the navigation arrows in the Selected Guest Banner.

Enhanced Touch Targets

Improved table interaction for touchscreens

Tables on mobile devices are optimized for touch interaction with larger minimum touch areas and visual feedback to make assignments quick and error-free.

Touch Optimizations

  • Minimum Touch Area: All tables have a minimum 60px × 60px touch target, meeting accessibility standards and making them easier to tap accurately.
  • No Double-Tap Zoom: Tables include touch-action: manipulation to prevent the iOS double-tap zoom delay, ensuring instant response.
  • Visual Feedback: When a guest is selected, available tables pulse with a subtle animation to indicate they are ready to receive the assignment.

Accessibility: These enhancements follow WCAG guidelines for touch target sizes, making the app more accessible for users with motor impairments.

Mobile Experience Summary

These mobile-optimized features work together to create a seamless guest assignment workflow on smartphones. By reducing context switching and providing touch-friendly controls, you can efficiently seat guests as they arrive without the friction of navigating between multiple views.

9. Creating Custom Layouts

Advanced users can create bespoke floor plans using the Layout Creator tool (LayoutCreator.html).

layout.json format Schema definition 
{
  "internalId": 1,
  "displayId": "Table 1",
  "capacity": 10,
  "orientation": "horizontal",
  "gridRow": 5,
  "gridCol": 2
}
PropertyDescription
internalIdUnique number (Never change once set).
orientation"horizontal" or "vertical" layout.
gridRow / gridColPosition in the layout grid (Starts at 0).
capacityNumber of seats per table (Default: 10).

10. Importing Data

Guest List Import

Ensure your Excel or CSV file contains exactly these column headers:

  • Host Name
  • Seats Required
  • Notes (optional)

Layout Import

  1. 01 Click Menu → Import Layout.
  2. 02 Select your custom .json file.
  3. 03 The map reloads with your new floor plan.

12. Troubleshooting

Guest import fails

Verify that your Excel/CSV has the correct column headers (Host Name, Seats Required).

Tables not displaying

Check the layout JSON for valid gridRow and gridCol coordinate values.
